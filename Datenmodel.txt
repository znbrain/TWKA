Datenmodell:
Für das Datenmodell soll das EntityFramework verwendet werden. die Klassen sind demnach entsprechend zu erstellen. 

Entitäten:
Wettkampf
- WettkampfID int
- Name string
- Datum datetime

Durchgang
- DurchgangID int
- WettkampfID int
- Nummer smallint
- Bezeichnung string

Verein
- VereinID int
- Name string
- Bezeichnung string

Altersklasse
- AltersklasseID int
- WettkampfID int
- WertungsklasseID int
- Name string
- Bezeichnung string

Leistungsklasse
- LeistungsklasseID int
- WettkampfID int
- Name string
- Bezeichnung string

Teilnehmer
- TeilnehmerID int
- WettkampfID int
- VereinID int
- RiegeID int
- AltersklasseID int
- LeistungsklasseID int
- Name string
- Vorname string

Riege
- RiegeID int
- WettkampfID int
- DurchgangID int
- Nummer smallint
- Bezeichnung string

Gerät
- GerätID int
- Name string
- Bezeichnung string

Station
- StationID int
- WettkampfID int
- GerätID int
- RiegeID int
- Reihenfolge smallint
- WertungErforderlich boolean

Wertung
- WertungID int
- WettkampfID int
- StationID int
- TeilnehmerID int
- DWert decimal(2,2)
- EWert decimal(2,2)
- PWert decimal(2,2)

Wertungsklasse
- WertungsklasseID int
- WettkampfID int
- Name string
- Bezeichnung string



Beziehungen:
Die Beziehungen sind immer nach dem Schema 1 -> N notiert
Wettkampf -> Durchgang
Wettkampf -> Riege
Wettkampf -> Station
Wettkampf -> Altersklasse
Wettkampf -> Leistungsklasse
Wettkampf -> Wertungsklasse
Wettkampf -> Teilnehmer
Wettkampf -> Wertung
Durchgang -> Riege
Riege -> Station
Riege -> Teilnehmer
Wertungsklasse -> Altersklasse
Wertungsklasse -> Leistungsklasse
Altersklasse -> Teilnehmer
Leistungsklasse -> Teilnehmer
Verein -> Teilnehmer
Teilnehmer -> Wertung
Station -> Wertung
Gerät -> Station
